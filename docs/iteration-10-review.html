7. צעדים הבאים — סדר עדיפויות

  • 1
    🎵 אוטומציית ייצור מוזיקה ב-Suno קריטי
    חקירת Suno API לגישה אוטומטית. התשתית מוכנה (pipeline + prompts + manifest). צריך: חשבון Suno Premier, API access, אוטומציה של copy prompt → generate → download → import.
  • 2
    🎤 ייצור Batch 1 — 10 טראקים ראשונים קריטי
    5 ערכות דמויות (Ki, Mika, Yuki, Sensei Zen, Glitch) + 3 gameplay (Practice, Speed Test, Tournament) + 2 story (Emotional, Victory). כל ה-prompts מוכנים ב-scripts/suno-prompts.json.
  • 3
    🎨 Expression Sheets חסרים גבוה
    יצירת expression sheets לדמויות שעדיין חסרות: Yuki, Kai, Pixel, Rex, Virus. שימוש ב-character-consistency skill כדי להבטיח עקביות עם model sheets קיימים. סנסיי זן צריך סט מלא (הנוכחי חלקי).
  • 4
    👩 עיצוב מחדש של מיקה — יותר נשית גבוה
    פידבק משתמש: מיקה נראית "tomboyish" מדי, בנות לא יתחברו אליה. צריך עיצוב מחדש יותר נשי/קלאסי. העיצוב הנוכחי יכול לעבור לדמות חדשה. שימוש ב-Visual Bible + Consistency Skill לעיצוב חדש.
  • 5
    🔊 אפקטי צליל על כל אינטראקציית UI בינוני
    הוספת SFX לכל כפתור, מעבר עמוד, הצלחה, כשלון, hover, ואינטראקציית משתמש. התשתית קיימת (Web Audio + MP3 SFX). צריך: מיפוי אירועים לצלילים + עטיפת כל הקומפוננטות.
  • 6
    🛡️ מנגנוני Anti-Frustration בינוני
    מחקר ומימוש מנגנונים למניעת תסכול: עידוד אוטומטי, התאמת קושי דינמית, hints על בקשה, מערכת עזרה נגישה. המשתמש אישר את כל המסקנות מ-game-ui-research.html.
  • 7
    🏆 פרוטוטייפ מערכת טורנירים בינוני
    טורנירים אוטומטיים מסונכרנים עם לוח שנה בית ספרי. הפרדת גילים, live vs time-limited, הפרדת dojos. צריך: backend schema, matchmaking logic, UI tournament bracket.
