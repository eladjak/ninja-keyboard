🎯 המלצת אסטרטגיה: Suno כמנוע ראשי
למה Suno, ואיך לשלב עם שאר הכלים.
מצב ה-API של Suno — מרץ 2026
  • אין API רשמי ציבורי — Suno לא מפרסם REST API פתוח עם API key
  • Suno MCP Server קיים — כבר מותקן אצלנו ב-C:\Users\eladj\suno-mcp-direct
  • suno-api (GitHub) — פרויקט open-source שעושה reverse-engineer ל-Suno עם Playwright + 2Captcha
  • API של צד שלישי — sunoapi.org ו-udioapi.pro מציעים REST API מנוהל (~$0.40-0.50/רצועה)
  • Cookie Auth — כל הפתרונות הלא רשמיים דורשים cookie מהדפדפן (מתעדכן כל ~30 ימים)
Approach A: Suno MCP (מומלץ)
  • כבר מותקן ועובד בפרויקט
  • אינטגרציה ישירה עם Claude Code
  • פקודות: suno_generate_track, suno_download_track
  • חצי-אוטומטי: Claude שולח prompt, מוריד תוצאה
  • מתאים ליצירת 5-8 רצועות ב-session אחת
Approach B: ידני דרך suno.com
  • גמישות מלאה — שליטה על כל פרמטר
  • Preview מיידי בדפדפן
  • Extend ו-Remix נוחים
  • מתאים לרצועות שדורשות fine-tuning
  • חלופה כש-MCP נשבר (cookie expires)
Approach C: Udio לעריכה (משלים)
  • Inpainting — שינוי סקשן ספציפי ברצועה
  • Stems — הפרדה לערוצים (drums, melody, bass)
  • שימושי לרצועות שהן "כמעט מושלמות"
  • Remix של רצועת Suno קיימת עם סגנון אחר
  • מחיר דומה ל-Suno
החלטה ממתינה: Suno Pro או Premier?
  • Pro ($10/חודש) — 2,500 קרדיטים = ~250 רצועות V4.5 / ~210 רצועות V5. מספיק ל-48 רצועות + ניסיונות.
  • Premier ($30/חודש) — 10,000 קרדיטים. מוגזם לפרויקט בודד, אלא אם רוצים גרסאות רבות לכל רצועה.
  • המלצה: Pro לחודש אחד ($10) — יספיק ל-48 רצועות + ~100 ניסיונות נוספים. סה"כ: $10.
🎨 הדיון הגדול: Chiptune/Pixel vs Cinematic
המשתמש מתלבט בין סגנון pixel/chiptune לבין סגנון קולנועי יותר. הנה הניתוח המלא.
Chiptune / 8-bit / Pixel
  • נוסטלגיה — צלילי 8-bit מייצרים חיבור רגשי מיידי
  • פשטות — מנגינות פשוטות וקליטות, קלות לזכור
  • BPM גבוה — 120-160 BPM טבעי, מתאים לאנרגיית הקלדה
  • לולאה מושלמת — chiptune עושה loop חלק בצורה טבעית
  • גודל קובץ קטן — MP3 קל, טעינה מהירה
  • עקביות — קל לשמור סגנון אחיד בין 48 רצועות
  • מותג מוכר — Main Theme V2 כבר chiptune מצוין!
מתאים ל: גילאי 6-12 (שתיל, נבט, גזע). רפרנסים: Brawl Stars, Minecraft, Undertale, Pokemon
Cinematic / Orchestral
  • רגש — מוזיקה אורקסטרלית מייצרת עומק רגשי
  • דרמה — מושלם ל-boss battles ו-climax
  • בגרות — נשמע "רציני" יותר לנוער (ענף, צמרת)
  • סיפור — מתאים ל-cutscenes ולרגעי narrative
  • Dynamic — שכבות אורקסטרליות מאפשרות adaptive music
מתאים ל: גילאי 12-16+ (ענף, צמרת) + boss battles. רפרנסים: Zelda, Genshin Impact, Final Fantasy
ההמלצה: גישה היברידית — Chiptune כבסיס + Cinematic לרגעי שיא
זה בדיוק מה שעושים המשחקים הכי מצליחים לילדים. Undertale הוא הדוגמה המושלמת: chiptune בזרימה הרגילה, אורקסטרה אפית ברגעי boss. Pokemon מתחיל chiptune ומתפתח לסינתטי. Brawl Stars משלב EDM עם chiptune.

שלב 1: הכנת Prompt
  1. בחר רצועה מרשימת ה-48 ב-soundtrack-master-plan
  2. קח את ה-prompt המוכן (כבר כתוב לכל רצועה)
  3. הוסף "anchor tags" לעקביות: ninja game, kids friendly, loopable
  4. ציין BPM מדויק (ראה טבלה בסקשן פרומפטים)
שלב 2: יצירה (Generate)
  1. צור 2 גרסאות לכל רצועה (Suno מייצר 2 בכל פעם)
  2. אם שתיהן לא טובות — שנה prompt ונסה שוב
  3. אם אחת "כמעט" — השתמש ב-Extend או שלח ל-Udio inpainting
  4. לרצועות background — השתמש ב-duration="long"
  5. לjingles — duration="short" (5-15 שניות)
שלב 3: Post-Processing (ffmpeg)
Normalize ל--14 LUFS
ffmpeg -i input.mp3 -af loudnorm=I=-14:TP=-1:LRA=11 output.mp3
Fade-in/Fade-out (loop חלק)
ffmpeg -i input.mp3 -af "afade=t=in:st=0:d=0.5,afade=t=out:st=END-0.5:d=0.5" output.mp3
Trim ל-loop point מדויק
ffmpeg -i input.mp3 -ss 0.1 -to 120.0 -c:a libmp3lame -q:a 2 output.mp3
שלב 4-5: בדיקת Loop + Deploy
בדיקת לולאה
  1. השמע ברצף 3 פעמים — האם יש "קפיצה" במעבר?
  2. בדוק ש-BPM עקבי (אין האצה/האטה)
  3. וודא שאין silence בסוף הקובץ
  4. אם לא חלק — trim עם ffmpeg
Deploy
  1. שמור ב-public/audio/music/[name].mp3
  2. עדכן music-manifest.json
  3. עדכן PROGRESS.md
  4. Commit: feat: add [track-name] music track

טיפ: יצירה "חכמה" — לא ליצור הכל ביום אחד
  • כל batch — session נפרדת. לא לערבב סגנונות שונים.
  • Batch 1 (jingles) + Batch 2 (battle) = יום אחד אפשרי.
  • Batch 3 (character themes) — צריך אוזן רעננה, לא באותו יום.
  • Batch 4-5 — רק אחרי שהפיצ'רים הרלוונטיים מפותחים.
  • בין batch ל-batch: עדכן manifest, בדוק loop, commit.

Crossfade בין מסכים
  • Equal-power curve: שימוש ב-Math.cos ו-Math.sin ליצירת עקומת fade חלקה
  • Duration: 2-3 שניות (configurable)
  • Pre-load: הרצועה הבאה נטענת מראש (buffer)
  • linearRampToValueAtTime: Web Audio scheduler
  • אותו BPM: רצועות באותו BPM = crossfade חלק יותר
Crossfade Implementation Pattern
// Equal-power crossfade const fadeDuration = 2.5; // seconds const now = audioContext.currentTime; // Fade out current currentGain.gain.setValueAtTime(1, now); currentGain.gain.linearRampToValueAtTime(0, now + fadeDuration); // Fade in next nextGain.gain.setValueAtTime(0, now); nextGain.gain.linearRampToValueAtTime(1, now + fadeDuration); // Stop old source after fade completes currentSource.stop(now + fadeDuration + 0.1);
Adaptive Music (עתידי)
  • Layer System: כל רצועה מחולקת לשכבות (drums, melody, bass)
  • Intensity: ככל ששחקן מקליד מהר, יותר שכבות מתווספות
  • קושי: BPM עולה עם רמת הקושי (stretch time playback)
  • Combo: שכבות נוספות מתווספות ב-combo streaks
  • דורש Stems: Udio export stems → שכבות נפרדות
Phase 2+ — דורש stems מ-Udio ופיתוח layer manager. לא ב-MVP.
