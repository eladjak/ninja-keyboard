8. Animation Always ON -- Updated Spec

Policy Change

OLD: Characters were hidden/static for older age themes (Anaf 12-14, Tzameret 14-16+).

NEW: Characters are ALWAYS animated at ALL age levels. The difference is controlled by a Settings toggle, not by age theme automatically.

Updated Age Matrix

Age ThemeCharacter SizeAnimationDialogue
Shatil (6-8)LARGE 120pxFull animation, reactions to every keystrokeFull bubbles, max 5 words
Nevet (8-10)MEDIUM 80pxFull animation, subtle idleFull bubbles, max 8 words
Geza (10-12)SMALL 48pxFull animation, minimal idleFull bubbles, 10+ words
Anaf (12-14)SMALL 36pxFull animation (NEW! was static)Short tooltips
Tzameret (14-16+)SMALL 28pxFull animation (NEW! was hidden)Minimal / optional

What Changes

  • All ages get animated characters by default -- no more auto-hiding
  • Older ages get SMALLER characters but they are still animated and present
  • A Settings page controls graphics level (see Section 9 below)
  • The user decides how much graphics to show, not the age theme
  • Teacher view remains professional (Sensei Zen + Pixel only, small)

Animation Performance Rules

  • All animations use transform/opacity ONLY (no width/height/top/left)
  • Idle animations: CSS-based, < 3KB per character
  • Reaction animations: Framer Motion, max 200ms for feedback
  • Expression changes: crossfade via opacity, 150ms transition
  • Lazy-load: only the active page character loads full assets

Implementation Notes

  • Store graphics level in settings-store (1-5, default 3)
  • CSS custom property: --graphics-level: 1|2|3|4|5

Gemini Generation Workflow

  1. Reference Image: Always attach an existing model sheet (e.g., ki-boy-v2.jpg) as style reference
  2. Batch by Character: Generate all expressions for one character in a single session for style consistency
  3. Aspect Ratios: Expression sheets = 3:2, Backgrounds = 16:9, Boss splash = 16:9, Portraits = 1:1, Unlock art = 2:3
  4. Output Path: public/images/characters/expressions/[character]-expressions.jpg
  5. Quality Check: Verify consistent style, correct colors, appropriate expressions before integration
  6. Tool: cd ~/.claude/skills/nano-banana-poster/scripts && node --loader ts-node/esm generate_poster.ts --aspect [ratio] "[prompt]"
