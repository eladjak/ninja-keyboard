פילוסופיית יריבי AI

העקרונות שמנחים את בניית המערכת
ליבה
💡
עיקרון מנחה: יריב AI טוב הוא כזה שהשחקן לא מבחין שהוא בוט. הוא מקליד כמו ילד בן גילו — עם טעויות, היסוסים, פרצי מהירות, ורגעי "אה, שכחתי את המקש".
🎭 לא מכונה — ילד
המטרה העיקרית: השחקן חייב להרגיש שהוא מתחרה נגד ילד אמיתי, לא נגד מחשב. אף ילד לא מקליד בקצב מושלם וקבוע. יריב AI שמכה כל אות בדיוק של 150ms הוא מפחיד וניכר כמזויף. במקום זה — תנודות, עצירות קטנות, טעויות ותיקונים.
🎯 Rubber-Banding — גומיית קושי
כמו במשחקי מרוצים, ה-AI מתאים את עצמו לרמת השחקן. אם השחקן מוביל — ה-AI מאיץ קצת. אם ה-AI רחוק מדי — הוא "טועה". המטרה: ה-AI מנצח 40-50% מהזמן. מספיק אתגר בלי תסכול.
🎨 אישיות ייחודית לכל יריב
כל דמות יריבה מקלידה בסגנון שונה לגמרי. Shadow (צל) יציב ומדויק. Storm (סערה) מהיר ובלתי צפוי. Blaze (להבה) מתחיל חם ונשרף. זה לא רק מספרים שונים — זה חוויה אחרת בכל קרב.
🫣 לעולם לא רמאות גלויה
כל התאמת קושי חייבת להיות בלתי נראית. ה-AI לא "קופץ" פתאום ב-20 WPM. הוא לא עוצר באמצע מילה. כל שינוי קצב מתרחש בצורה טבעית — דרך "טעות" שנראית אמיתית, או "פרץ מהירות" שנראה כמו מוטיבציה.

מנוע
🔬
מבוסס מחקר: מודל ההקלדה מבוסס על מחקרים אמפיריים על דפוסי הקלדה של ילדים בני 6-16. תזמון בין-מקשי עוקב התפלגות גאוסיאנית, לא אחידה.

תזמון בין-מקשי (Inter-Key Timing)

הקלדה אנושית לעולם לא מטרונומית. הזמן בין כל שני מקשים משתנה בצורה גאוסיאנית סביב ממוצע שנקבע לפי רמת הקושי.

דפוסי טעויות אנושיות

טעויות הן המפתח לאמינות. ה-AI עושה את אותם סוגי טעויות שילדים אמיתיים עושים בהקלדה בעברית:

🔤 שגיאת מקש שכן

לחיצה על מקש סמוך במקלדת העברית. לדוגמה: ש במקום ד, או כ במקום ל. הסיכוי עולה ככל שהמהירות גדלה.

סיכוי:
35%

🐌 היסוס באותיות נדירות

אותיות סופיות (ץ, ף, ך, ם, ן) גורמות להאטה של 1.5-3x בזמן התגובה. ילדים צעירים צריכים "לחפש" אותן.

סיכוי:
60%

⚡ פרצי מהירות

מילים מוכרות (כמו "של", "את", "הוא") מוקלדות מהר בהרבה מהממוצע — כמו שריר-זיכרון. מילים חדשות או ארוכות גורמות להאטה.

סיכוי:
45%

⌫ תיקונים לא מיידיים

כשה-AI טועה, הוא לא מתקן ב-0ms. יש עיכוב של 200-800ms לפני לחיצה על Backspace — כמו שילד שם לב לטעות ואז מוחק.

סיכוי:
80%

התנהגויות זמן

התנהגות תיאור פרמטרים אפקט
חימום (Warmup) 10 השניות הראשונות איטיות יותר מכפיל 1.3-1.6x על זמן בין מקשים WPM נמוך יותר בהתחלה
עייפות (Fatigue) האטה הדרגתית אחרי 60+ שניות +0.5% לזמן כל 10 שניות ירידה של 5-15% WPM בסופו של דבר
הפסקות חשיבה עצירות קצרות בין מילים 300-1200ms, בהסתברות 8-15% נראה כמו "חושב על המילה הבאה"
אזור נוחות מילים מוכרות = מהירות גבוהה מכפיל 0.6-0.8x למילים נפוצות פרצי מהירות טבעיים
ריכוז מחודש אחרי טעות — 2-3 אותיות איטיות מכפיל 1.2x, יורד הדרגתית מרגיש "נזהר יותר" אחרי טעות

5 רמות קושי בסיסיות

רמה עולם WPM מטרה שגיאות AI rubber-band יחס ניצחון מטרה
1 - שתיל גן קסום 10-20 8-12% חזק 60% שחקן
2 - נבט יער הרפתקאות 20-32 6-9% חזק 55% שחקן
3 - גזע הר הנינג'ה 32-45 4-7% בינוני 50% שחקן
4 - ענף ארמון הקוד 45-60 3-5% חלש 45% שחקן
5 - צמרת שמי האינסוף 60-80+ 2-3% מינימלי 40% שחקן

אלגוריתם Rubber-Banding

המערכת עוקבת אחרי פער התווים בין השחקן לבין ה-AI בזמן אמת, ומתאימה בהתאם:

🏃 שחקן מוביל (>10 תווים)

ה-AI מאיץ בהדרגה:

  • הפחתה של 5-15% מזמן בין מקשים
  • הפחתת סיכוי שגיאות ב-20%
  • הפחתת הפסקות חשיבה
  • לעולם לא יותר מ-+15% מ-WPM הבסיסי

🤖 AI מוביל (>15 תווים)

ה-AI "טועה" בטבעיות:

  • הוספת שגיאת מקש שכן
  • "הפסקת חשיבה" של 500-1500ms
  • האטה הדרגתית (לא פתאומית!)
  • לעולם לא נראה מכוון

בונה מתח — הסיום

כשנותרו פחות מ-20% מהטקסט, המערכת נכנסת למצב "מתח":

🔥 Tension Builder
  • שני הצדדים מאיצים — ה-AI מעלה WPM ב-10%, גם אם מוביל
  • הפער מתכווץ — אם יש פער גדול, הוא מצטמצם בהדרגה
  • אפקט ויזואלי — המסך מזדעזע קלות, מוזיקה מתגברת
  • סיום צמוד — מטרה: הפרש של 0-5 תווים בסיום (ב-70% מהמקרים)
  • אף פעם לא מופגן — השחקן חייב להרגיש שהוא ניצח/הפסיד "באמת"

מעקב רמה אדפטיבי

interface PlayerPerformanceTracker {
  // Rolling average over last 10 sessions
  rollingWPM: number
  rollingAccuracy: number

  // Win rate against AI (target: 40-50%)
  recentWinRate: number  // last 20 matches

  // Trend detection
  isImproving: boolean   // WPM increased 3+ sessions in a row
  isStruggling: boolean  // Win rate < 30% over last 10

  // Adjustment multiplier (0.7 = much easier, 1.3 = much harder)
  difficultyMultiplier: number
}

// Adjustment rules:
// - Win rate > 60% for 5+ matches → increase multiplier by 0.05
// - Win rate < 35% for 5+ matches → decrease multiplier by 0.05
// - Player rage-quit (left mid-match 2+ times) → decrease by 0.1
// - Player replayed same level 3+ times → decrease by 0.05
// - Player won 3 in a row easily (>15 char lead) → increase by 0.08

interface AITypingConfig {
  // Base typing speed
  baseWPM: number                    // Target words per minute
  wpmVariance: number                // ±% variance from base (e.g., 0.15 = ±15%)

  // Error behavior
  errorRate: number                  // 0-1, probability of error per keystroke
  correctionDelay: [number, number]  // [min, max] ms before pressing backspace
  errorType: 'adjacent' | 'random' | 'skip' // Most common error type

  // Rhythm
  burstProbability: number           // 0-1, chance of speed burst on familiar words
  burstSpeedMultiplier: number       // e.g., 1.5 = 50% faster during burst
  pauseProbability: number           // 0-1, chance of thinking pause between words
  pauseDuration: [number, number]    // [min, max] ms for pauses

  // Time effects

  // Character-specific behavior
  specialBehavior?: 'steady' | 'burst-pause' | 'burnout' | 'accelerate' | 'chaotic'

function calculateNextKeystrokeDelay(
  config: AITypingConfig,
  state: AITypingState,
  targetText: string,
  currentIndex: number
): { delay: number; char: string; isError: boolean } {

  // 1. Base delay from WPM
  //    Average chars per word in Hebrew ≈ 5
  const baseDelay = 60_000 / (config.baseWPM * 5)

  // 2. Gaussian jitter (Box-Muller transform)
  const jitter = gaussianRandom(0, baseDelay * config.wpmVariance)
  let delay = baseDelay + jitter

  // 3. Warmup effect (first N seconds)
  const elapsed = Date.now() - state.startTime
  if (elapsed < config.warmupDuration * 1000) {
    const warmupProgress = elapsed / (config.warmupDuration * 1000)
    const warmupFactor = config.warmupMultiplier - (config.warmupMultiplier - 1) * warmupProgress
    delay *= warmupFactor
  }

  // 4. Fatigue effect (after 60s)
  if (elapsed > 60_000) {
    const fatigueMinutes = (elapsed - 60_000) / 60_000
    delay *= (1 + config.fatigueRate * fatigueMinutes)
  }

  // 5. Special behavior modifiers
  delay = applySpecialBehavior(config, state, delay, elapsed)

  // 6. Word familiarity (common Hebrew words = faster)
  const currentWord = getCurrentWord(targetText, currentIndex)
  if (COMMON_HEBREW_WORDS.has(currentWord)) {
    if (Math.random() < config.burstProbability) {
      delay *= (1 / config.burstSpeedMultiplier)
    }
  }

  // 7. Final letter hesitation (sofit letters)
  const targetChar = targetText[currentIndex]
  if (SOFIT_LETTERS.has(targetChar)) {
    delay *= randomBetween(1.5, 3.0)
  }

  // 8. Thinking pause between words
  if (targetChar === ' ' && Math.random() < config.pauseProbability) {
    delay += randomBetween(...config.pauseDuration)
  }

  // 9. Rubber-banding adjustment
  delay = applyRubberBanding(config, state, delay)

  // 10. Error check
  const isError = Math.random() < config.errorRate
  let char = targetChar

  if (isError) {
    char = getErrorChar(targetChar, config.errorType)
    // After error: schedule correction
    state.pendingCorrection = {
      delay: randomBetween(...config.correctionDelay),
      correctChar: targetChar,
    }
  }

  // 11. Post-error caution (slower for 2-3 chars after correction)
  if (state.recentlyCorrected > 0) {
    delay *= 1.2
    state.recentlyCorrected--
  }

  return { delay: Math.max(delay, 30), char, isError }
}

// Gaussian random using Box-Muller transform
function gaussianRandom(mean: number, stdDev: number): number {
  const u1 = Math.random()
  const u2 = Math.random()
  const z = Math.sqrt(-2 * Math.log(u1)) * Math.cos(2 * Math.PI * u2)
  return mean + z * stdDev
}

התנהגויות מיוחדות

function applySpecialBehavior(
  config: AITypingConfig,
  state: AITypingState,
  delay: number,
  elapsed: number
): number {
  switch (config.specialBehavior) {
    case 'steady':
      // Minimize variance — clamp jitter
      return clamp(delay, delay * 0.85, delay * 1.15)

    case 'burst-pause':
      // Storm: random bursts then pauses
      if (state.inBurst) {
        delay *= 0.5  // Very fast during burst
        state.burstRemaining--
        if (state.burstRemaining <= 0) state.inBurst = false
      } else if (Math.random() < 0.08) {
        // Start a new burst (3-8 chars)
        state.inBurst = true
        state.burstRemaining = randomInt(3, 8)
        delay *= 0.5
      }
      return delay

    case 'burnout':
      // Blaze: linear decay over time
      const burnoutFactor = 1 + (elapsed / 60_000) * 0.04
      return delay * burnoutFactor

    case 'accelerate':
      // Virus: exponential speedup over time
      const accelFactor = Math.max(0.4, 1 - (elapsed / 120_000) * 0.6)
      return delay * accelFactor

    case 'chaotic':
      // Bug: high variance, sometimes very fast sometimes very slow
      const chaos = Math.random()
      if (chaos < 0.3) return delay * 0.5      // Very fast
      if (chaos < 0.5) return delay * 1.8      // Very slow
      return delay                               // Normal

    default:
      return delay
  }
}

import { useEffect, useRef, useCallback } from 'react'
import { motion, AnimatePresence } from 'framer-motion'
import { useAIOpponentStore } from '@/stores/ai-opponent-store'

interface AIOpponentProps {
  rivalId: 'shadow' | 'storm' | 'blaze' | 'yuki' | 'bug' | 'virus'
  targetText: string
  difficulty: number          // 1-5
  rubberBandEnabled: boolean  // false for tournaments
  onComplete: (result: AIMatchResult) => void
}

export function AIOpponent({
  rivalId,
  targetText,
  difficulty,
  rubberBandEnabled,
  onComplete,
}: AIOpponentProps) {
  const workerRef = useRef<Worker | null>(null)
  const {
    aiPosition,
    aiKeystrokes,
    aiWPM,
    setAIKeystroke,
    setAIComplete,
  } = useAIOpponentStore()

  // Initialize Web Worker
  useEffect(() => {
    workerRef.current = new Worker(
      new URL('@/workers/ai-typing-worker.ts', import.meta.url)
    )

    workerRef.current.onmessage = (event) => {
      const { type, payload } = event.data

      switch (type) {
        case 'keystroke':
          setAIKeystroke(payload)
          break
        case 'complete':
          setAIComplete(payload)
          onComplete(payload)
          break
      }
    }

    // Start AI typing
    workerRef.current.postMessage({
      type: 'start',
      payload: { rivalId, targetText, difficulty, rubberBandEnabled },
    })

    return () => workerRef.current?.terminate()
  }, [rivalId, targetText, difficulty])

  // Send player position updates to worker (for rubber-banding)
  const updatePlayerPosition = useCallback((pos: number) => {
    workerRef.current?.postMessage({
      type: 'playerUpdate',
      payload: { position: pos },
    })
  }, [])

  return (
    <div className="relative">
      {/* AI character avatar with reaction animations */}
      <RivalAvatar rivalId={rivalId} wpm={aiWPM} />

      {/* AI typing display */}
      <div className="font-mono text-sm leading-relaxed">
        {targetText.split('').map((char, i) => (
          <motion.span
            key={i}
            className={
              i < aiPosition
                ? 'text-green-400'
                : i === aiPosition
                  ? 'text-white bg-purple-600/30'
                  : 'text-gray-600'
            }
            animate={
              i === aiPosition
                ? { scale: [1, 1.05, 1], transition: { duration: 0.1 } }
                : {}
            }
          >
            {char}
          </motion.span>
        ))}
      </div>
    </div>
  )
}

Web Worker: ai-typing-worker

// src/workers/ai-typing-worker.ts
import { RIVAL_CONFIGS } from '@/lib/ai/rival-configs'
import { AITypingEngine } from '@/lib/ai/ai-typing-engine'

let engine: AITypingEngine | null = null

self.onmessage = (event) => {
  const { type, payload } = event.data

  switch (type) {
    case 'start': {
      const config = RIVAL_CONFIGS[payload.rivalId]
      const scaledConfig = scaleToDifficulty(config, payload.difficulty)

      engine = new AITypingEngine(
        scaledConfig,
        payload.targetText,
        payload.rubberBandEnabled
      )

      engine.onKeystroke = (event) => {
        self.postMessage({ type: 'keystroke', payload: event })
      }

      engine.onComplete = (result) => {
        self.postMessage({ type: 'complete', payload: result })
      }

      engine.start()
      break
    }

    case 'playerUpdate': {
      // Update rubber-banding with player's current position
      engine?.updatePlayerPosition(payload.position)
      break
    }

    case 'stop': {
      engine?.stop()
      engine = null
      break
    }
  }
}

function scaleToDifficulty(
  config: AITypingConfig,
  difficulty: number
): AITypingConfig {
  // Scale base WPM and error rate based on difficulty tier
  const wpmScale = [0.5, 0.7, 1.0, 1.2, 1.5][difficulty - 1]
  const errorScale = [2.0, 1.5, 1.0, 0.7, 0.5][difficulty - 1]

  return {
    ...config,
    baseWPM: config.baseWPM * wpmScale,
    errorRate: Math.min(config.errorRate * errorScale, 0.35),
  }
}

שימוש ב-Web Worker

  • כל חישובי התזמון רצים ב-Worker — לא חוסמים את ה-main thread
  • תקשורת דרך postMessage — רק אירועי הקשה (קטנים) עוברים
  • setTimeout ב-Worker — דיוק גבוה יותר מ-main thread
  • Drift compensation — תיקון הצטברות שגיאות תזמון

עלות CPU מינימלית

  • אין לולאת רנדור — רק עדכון DOM בעת הקשה
  • motion.span מותנה — אנימציה רק על תו נוכחי
  • זיכרון: ~2KB לכל מצב AI פעיל
  • מטרה: 60fps יציב גם במכשירים חלשים
