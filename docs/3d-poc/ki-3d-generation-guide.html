🔵 מדריך Tripo AI (tripo3d.ai)

1 הירשם לחינם

גש ל-tripo3d.ai → Create Account (Google/GitHub)

2 העלה תמונה (Image to 3D)

לחץ "Create" → "Image to 3D" → העלה את ki-boy.jpg (ה-Model Sheet המלא)

3 הגדר סגנון

אם יש אפשרות סגנון, בחר: Cartoon / Stylized / Chibi

הדבק את הפרומפט הקצר אם יש שדה טקסט

4 צור והמתן (~2-5 דקות)

לחץ Generate. צפה בתצוגה מקדימה. אם לא מספק — נסה שוב עם ה-front view בלבד

5 Rigging (אם זמין)

לחץ על "Rig" או "Animate" → בחר humanoid skeleton → Apply

6 הורד GLB

לחץ Download → בחר פורמט GLB

שמור בנתיב: public/characters/3d/ki-tripo.glb

🟢 מדריך Meshy AI (meshy.ai)

1 הירשם לחינם

גש ל-meshy.ai → Sign Up (Google/GitHub)

2 בחר Image to 3D

לחץ "Create" → "Image to 3D" → העלה את ki-boy.jpg

3 הגדרות

Art Style: Cartoon | Topology: Quad (אם זמין)

הדבק את הפרומפט הקצר בשדה Description

4 צור והמתן (~30 שניות - 3 דקות)

Meshy מהיר יותר מ-Tripo. תקבל תצוגה מקדימה 3D

5 Auto-Rig + Animation

לחץ על "Rig & Animate" → Meshy ייצור skeleton אוטומטי

בחר אנימציות: idle, wave, victory dance

⚠️ ב-free tier: 20 אנימציות בלבד

6 הורד GLB

לחץ Download → GLB

שמור בנתיב: public/characters/3d/ki-meshy.glb

הגבלת Free Tier: Meshy-4 מודלים בלבד להורדה בחינם. Meshy-5/6 דורשים Pro ($20/חודש). נסה קודם ב-free — אם האיכות לא מספיקה, שקול Pro לחודש אחד.

💃 Mixamo — אנימציות חינם

בין אם יצרת ב-Tripo או Meshy, Mixamo מוסיף אנימציות איכותיות בחינם

1 גש ל-Mixamo

mixamo.com → Sign in with Adobe ID (חינם)

2 העלה את המודל

"Upload Character" → העלה את קובץ ה-GLB או FBX שהורדת

Mixamo יזהה אוטומטית את הגוף וייצור skeleton

3 בחר אנימציות

חפש וסמן את האנימציות האלה:

Moodחיפוש ב-Mixamoהערות
idle"Idle" / "Breathing Idle"לולאה, נשימה עדינה
typing"Typing" / "Sitting Typing"יש אנימציית הקלדה!
celebrate"Victory" / "Happy Dance" / "Cheering"ניצחון, ריקוד
error"Disappointed" / "Sad Idle" / "Defeat"תגובת טעות
4 הורד כל אנימציה

Format: FBX Binary | Skin: With Skin (לאנימציה הראשונה) / Without Skin (לשאר)

Keyframe Reduction: None

5 המר ל-GLB (Blender)

1. פתח Blender → Import FBX

2. ודא שהאנימציה נטענה (תראה ב-Dope Sheet)

3. Export → glTF 2.0 (.glb) עם "Include Animations" מסומן

4. שמור כ: public/characters/3d/ki.glb

💡 טיפ: ב-Blender ניתן לייבא את כל ה-FBX קבצים לסצנה אחת ולייצא GLB אחד עם כל האנימציות. כך ה-R3F viewer יכול לעבור בין אנימציות לפי mood.

🎮 מיפוי אנימציות → קוד

הקוד ב-ki-model.tsx כבר מחפש אנימציות לפי שם:

Animation Name Convention 
// Current code in ki-model.tsx:
const animationName = names.find(
  (n) => n.toLowerCase().includes(mood)
       || n.toLowerCase().includes('idle')
)

// Name your animations in Blender like:
// "idle"     → matches mood='idle'
// "typing"   → matches mood='typing'
// "celebrate" → matches mood='celebrate'
// "error"    → matches mood='error'

אם שמות האנימציות מ-Mixamo שונים (למשל "Victory Dance" במקום "celebrate"), פשוט שנה את שמם ב-Blender לפני הייצוא.

✅ צ'קליסט איכות (לפני שימוש בפרודקשן)

  • המודל נראה כמו קי (צבעים, בגדים, שיער, רצועת ראש)
  • פרופורציות צ'יבי (ראש גדול ~40% מהגוף)
  • אין ארטיפקטים בטקסטורה
  • Rigging תקין — הגוף זז טבעי
  • 4 אנימציות קיימות (idle, typing, celebrate, error)
  • GLB פחות מ-5MB (אידיאלי: 1-3MB)
  • נטען ב-POC page בלי שגיאות
  • סיבוב אוטומטי עובד
  • מעבר בין moods חלק (fadeIn/fadeOut)

🚀 שלבים הבאים (אחרי שיש מודל)

1 בדיקת POC

הכנס את ה-GLB ל-public/characters/3d/ki.glb → פתח /3d-poc → בדוק את 4 ה-moods

2 אינטגרציה בדף הבית

החלף את ה-2D avatar של קי בדף הבית ב-Ki3DViewer (כבר קיים כ-component)

3 Typing Feedback

חבר את ה-mood ל-typing engine: הקלדה נכונה → celebrate, שגיאה → error, ממתין → idle

4 Performance Budget

ודא שה-GLB < 3MB. אם גדול: השתמש ב-gltf-transform ל-compress

npx @gltf-transform/cli optimize ki.glb ki-opt.glb --compress draco
